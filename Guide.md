# Bug Detactor 사용 가이드

Slack 채널에서 버그 이슈를 분석하고, 코드 수정 및 PR 생성까지 할 수 있는 봇입니다.

---

## 기본 흐름

```
이슈 등록 → 관련 레포 추천 → 코드 분석 → 코드 수정 → PR 생성
```

모든 작업은 **Slack 스레드 안에서** 진행됩니다.

---

## Step 1. 이슈 등록

품질팀이 Slack 채널에 버그 이슈를 작성합니다.

> 예시: "고객 채팅 화면에서 새로고침하면 이전 대화 내역이 사라집니다"

---

## Step 2. 관련 레포 찾기

이슈가 어느 레포와 관련 있는지 모를 때 사용합니다. 두 가지 방법이 있습니다.

### 방법 A: 이모지 반응

이슈 메시지에 :eyes: (👀) 이모지를 달면 봇이 자동으로 관련 레포를 추천해줍니다.

### 방법 B: analyze 명령어 (레포 미지정)

이슈 스레드에 다음과 같이 입력합니다:

```
analyze: 채팅 히스토리 로딩 관련 코드 확인해줘
```

봇이 서비스 목록을 보고 관련 레포를 추천해줍니다.

---

## Step 3. 코드 분석

추천받은 레포를 지정해서 실제 코드 분석을 요청합니다.

```
analyze: dworks-cstalk: 게스트 채팅 히스토리 로딩 로직 분석해줘
```

**형식:** `analyze: 레포명: 지시내용`

봇이 해당 레포의 코드를 읽고 다음 형식의 분석 리포트를 스레드에 올려줍니다:
- 이슈 요약
- 원인 분석 (관련 파일/함수 포함)
- 수정 필요 범위
- 완료 조건

여러 레포를 한번에 분석할 수도 있습니다:

```
analyze: dworks-cstalk, attic-core-asset-messaging: 메시지 유실 원인 분석해줘
```

---

## Step 4. 코드 수정

분석 결과를 확인한 뒤 수정을 요청합니다.

```
fix: dworks-cstalk: 게스트 채팅 히스토리 페이징 로직에서 offset 계산 버그 수정해줘
```

**형식:** `fix: 레포명: 지시내용`

봇이 코드를 수정한 뒤 **파일별 diff**를 스레드에 올려줍니다.

여러 레포를 동시에 수정할 수도 있습니다:

```
fix: dworks-cstalk, attic-core-asset-buzzer: WebSocket 재연결 타임아웃 30초로 변경
```

---

## Step 5. PR 생성

diff를 확인하고 문제가 없으면 스레드에 PR 생성을 요청합니다.

### 브랜치명 지정 (권장)

```
PR 요청해줘 DWFLOW-1234
```

→ `develop` 기반으로 `DWFLOW-1234` 브랜치를 만들고 PR을 생성합니다.

### 브랜치명 생략

```
PR 요청해줘
```

→ `develop` 기반으로 `DWFLOW-FIX-{랜덤4자리}` 브랜치를 자동 생성합니다. (예: `DWFLOW-FIX-3847`)

봇이 자동으로 브랜치 생성 → commit → push → PR 생성을 수행하고, PR 링크를 알려줍니다.

> PR 타겟 브랜치는 항상 `develop`입니다.
> PR 생성은 바로 직전 `fix:` 작업에서 수정한 레포들을 대상으로 합니다.

---

## 명령어 요약

| 명령어 | 설명 |
|--------|------|
| 👀 이모지 반응 | 관련 레포 추천 |
| `analyze: 지시내용` | 관련 레포 추천 (이모지와 동일) |
| `analyze: 레포명: 지시내용` | 해당 레포 코드 분석 리포트 |
| `fix: 레포명: 지시내용` | 코드 수정 + diff 게시 |
| `PR 요청해줘 DWFLOW-1234` | 지정 브랜치명으로 PR 생성 (→ develop) |
| `PR 요청해줘` | 랜덤 브랜치명으로 PR 생성 (→ develop) |

---

## 주의사항

- 모든 명령어는 **이슈 메시지의 스레드** 안에서 입력해야 합니다. 채널에 직접 쓰면 동작하지 않습니다.
- `fix:` 전에 `analyze:`로 먼저 분석하는 것을 권장합니다.
- PR 생성은 `fix:` 작업 이후에만 가능합니다. `fix:` 없이 `PR 요청해줘`를 입력하면 오류가 납니다.
- 봇이 응답하지 않으면 봇이 해당 채널에 초대되어 있는지 확인하세요. (`/invite @봇이름`)
